🌐 Siga os passos abaixo para colocar o WMS online com dados na nuvem. Tempo estimado: 15 minutos.
⚙️
Firebase: não configurado
Configure seguindo os passos abaixo

PASSO 1 — Criar projeto no Firebase (banco de dados gratuito)

  1. Acesse https://console.firebase.google.com e entre com sua conta Google
  2. Clique em "Adicionar projeto" → nomeie como etex-wms → clique em Continuar
  3. Desative o Google Analytics (não precisa) → "Criar projeto"
  4. No menu lateral, clique em Firestore Database → "Criar banco de dados"
  5. Escolha "Iniciar no modo de teste" → selecione região southamerica-east1 → Ativar

PASSO 2 — Pegar as credenciais do Firebase

  1. No console Firebase, clique na engrenagem ⚙️ → "Configurações do projeto"
  2. Role para baixo até "Seus aplicativos" → clique em </> Web
  3. Registre com o nome etex-wms-web → clique em "Registrar app"
  4. Copie o objeto firebaseConfig que aparece — ele tem este formato:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "etex-wms.firebaseapp.com",
  projectId: "etex-wms",
  storageBucket: "etex-wms.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123..."
};

PASSO 3 — Configurar as credenciais aqui no WMS

Cole cada valor do seu firebaseConfig nos campos abaixo:

PASSO 4 — Ativar Autenticação por E-mail no Firebase

  1. No console Firebase → menu lateral → Authentication
  2. Clique em "Começar" → aba "Sign-in method"
  3. Clique em E-mail/senha → ative o primeiro toggle → Salvar

PASSO 5 — Regras do Firestore (permitir acesso autenticado)

  1. No console Firebase → Firestore Database → aba Regras
  2. Substitua o conteúdo pelas regras abaixo e clique em Publicar:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}

✅ Estas regras exigem login — só usuários autenticados acessam os dados.

PASSO 6 — Criar o primeiro Administrador

  1. No console Firebase → Authentication → aba Users → "Add user"
  2. Informe seu e-mail e uma senha forte → Add user
  3. Copie o UID gerado (coluna User UID)
  4. Vá em Firestore Database → "Start collection" → ID: usuarios
  5. Document ID = o UID copiado. Adicione os campos:
nome: "Seu Nome" (string)
email: "seu@email.com" (string)
role: "admin" (string)
status: "ativo" (string)
criadoEm: "2025-01-01T00:00:00.000Z" (string)

Depois de logado como admin, crie os demais usuários diretamente pelo menu 👥 Usuários dentro do WMS.

PASSO 7 — Publicar no GitHub Pages

  1. Acesse https://github.com e entre na sua conta
  2. Clique em "New repository" → nome: etex-wms → público → Create
  3. Na página do repositório, clique em "uploading an existing file"
  4. Arraste o arquivo baixado no Passo 8 → Commit changes
  5. Vá em Settings → Pages → Source: Deploy from branch → Branch: main → Save
  6. Aguarde ~2 minutos. Seu WMS estará em:
    https://[seu-usuario].github.io/etex-wms/wms_etex_gypsum.html

⚠️ Precisará adicionar o domínio do GitHub Pages como "domínio autorizado" no Firebase:
Console Firebase → Authentication → Settings → Authorized domains → Add domain → [seu-usuario].github.io
